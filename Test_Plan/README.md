# Test Plan – E-commerce Web Application

## Project Name
ShopEasy – E-commerce Web Application

## Objective
The objective of this test plan is to validate the functionality,
usability, and reliability of the E-commerce web application using
manual testing techniques.

## Scope

### In Scope
- User Registration
- User Login & Logout
- Product Search
- Product Details Page
- Add to Cart
- Checkout
- Payment (Dummy/Test Mode)
- Order Confirmation

### Out of Scope
- Automation Testing
- Performance Testing
- Security Testing
- Real Payment Gateway Testing

## Test Approach
- Manual testing will be performed
- Test cases will be written based on requirements
- Defects will be logged and tracked
- Regression testing will be done after fixes

## Test Types
- Functional Testing
- UI Testing
- Smoke Testing
- Regression Testing

## Test Environment
- Browser: Google Chrome
- OS: Windows
- Testing Type: Manual

## Entry Criteria
- Application should be accessible
- Requirements should be available
- Test environment should be ready

## Exit Criteria
- All planned test cases are executed
- Critical defects are fixed or reported
- Test summary report is prepared

## Roles and Responsibilities
- Tester: Test case design, execution, defect reporting
- Developer: Fix defects
- Manager: Review and approval

## Risks and Assumptions
- Requirement changes may impact testing
- Limited test data availability
